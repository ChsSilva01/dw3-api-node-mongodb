##### **APIs REST**

*//Arquitetura REST, basicamente ela utiliza de métodos de páginas web's (http: GET, POST, PUT, DELETE)*



status code: 200



> Formato JSON

// meio de se entregar informações por meio de um grupo de array



GET: recuperar informações;

POST: enviar dados;

PUT: substitui;

DELETE: deleta.



200 - (OK): foi bem sucedida;

201 - (CREATED): foi bem sucedida e algo foi criado nesse meio;

204 - (NO CONTENT): indica que foi bem sucedida porém não está retornando nada (não necessariamente precisaria retornar algo);

400 - (BAD REQUEST): as informações enviadas são inválidas seja pelos dados mal formados, seja pela ausência dos mesmos;

404 - (NOT FOUND): Indica que o recurso não foi encontrado;

500 - (INTERNAL SERVER EROR): deu errado porém não sabe dizer aonde exatamente.
